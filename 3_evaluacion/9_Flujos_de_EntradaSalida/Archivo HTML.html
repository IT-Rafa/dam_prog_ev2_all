Lectura y escritura de un fichero

Datos generales

Stream

Lista de datos que se pasan de un lugar a otro

Stream I/O

Lista de datos que se pasan de un lugar externo del programa (archivos, datos en red (sockets), etc.) al propio programa

flujo de bytes

Se refiere a un Stream I/O que nos envía los datos directamente en bytes, que es la forma en que se graban los datos mínimos en el ordenador. Este sistema es útil con archivos que no sean de texto, en general archivos binarios, como archivos de imagen, ejecutables binarios, sonido, etc.

flujo de caracteres

Se refiere a un Stream I/O que nos envía los datos organizados según una codificación de caracteres, donde un carácter no es necesariamente de un byte, sino que dependerá de su organización. (La codificación en un byte suele coincidir con los datos ASCII)

Este sistema es útil con archivos que sean de texto. Por defecto, si no se indica un código, se usará UTF-8, que es el sistema usado en el S.O. Linux, la web, Java y recomendado en general.

Clases

FileReader

Clase pensada para acceder al flujo de caracteres de un archivo del sistema, leyéndolos de carácter en carácter. Al heredar de InputStreamReader usa un bucle para traducir los bytes a caracteres, pero no para leer varios caracteres a la vez.

FileInputStream

Clase pensada para acceder al flujo de bytes de un archivo del sistema. (U)
